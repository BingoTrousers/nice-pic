## Architecture

**nice pic** is a booru-style photo imageboard. It is a TanStack Start (SSR) app using file-based routing, React Query, Tailwind CSS v4, and shadcn/ui (New York style, slate base).

### Data layer

All content lives in `src/data/posts.ts` as a static array of `Post` objects (`id`, `title`, `src`, `tags[]`). Helper functions (`filterByTags`, `searchPosts`, `getTagCounts`, `getAllTags`, `getPost`) are co-located there. There is no database or API — to add/remove posts, edit that file directly.

### Routing

TanStack Start uses **file-based routing** under `src/routes/`. `routeTree.gen.ts` is auto-generated by the Vite plugin — never edit it by hand. The root layout is `src/routes/__root.tsx` (wraps every page; contains `<SiteHeader />` and `<Outlet />`).

| File | URL |
|---|---|
| `routes/index.tsx` | `/` — post grid with tag filtering, search, pagination |
| `routes/post.$id.tsx` | `/post/:id` — single post detail |
| `routes/tags.tsx` | `/tags` — full tag browser |

Route search params are validated with Zod via `@tanstack/zod-adapter`. The index route carries `tags`, `mode`, `page`, and `q` params in the URL; pagination page size is 4 (`PAGE_SIZE` constant in `routes/index.tsx`).

### Key components

- `SiteHeader` — sticky nav with search form and dark mode toggle (persists via `localStorage`). Dark mode is applied by toggling the `dark` class on `<html>`.
- `PostGrid` — image grid rendered from the filtered/paginated post list.
- `TagSidebar` — tag filter panel with all/any mode toggle.
- `src/components/ui/` — shadcn/ui primitives; add new ones with `bunx shadcn@latest add <name>`.

### Vite config

`vite.config.ts` delegates entirely to `@lovable.dev/vite-tanstack-config`. **Do not manually add** TanStack Start, React, Tailwind, tsconfig-paths, or Nitro plugins — they are already included and duplicating them breaks the build. Pass extra config through `defineConfig({ vite: { ... } })` if needed.

### Lovable integration

This project is connected to [Lovable](https://lovable.dev). Do not force-push, rebase, amend, or squash commits that are already pushed — it rewrites history on Lovable's side. Keep the `main` branch in a working state at all times.
